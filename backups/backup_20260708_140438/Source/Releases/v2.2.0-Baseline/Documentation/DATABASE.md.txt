# Recruitment Automation Suite (RAS)

# DATABASE DOCUMENTATION

**Version:** 2.2.0 Baseline

---

# Database Overview

The Recruitment Automation Suite uses **Microsoft Excel** as its primary database.

Each worksheet represents a business entity within the recruitment lifecycle.

---

# Database Relationships

```text
Candidates_Master
      │
      ├── Resources
      ├── Resumes
      ├── Offers
      ├── Interviews
      ├── BGV
      ├── Deviation
      ├── NATS
      ├── NAPS
      ├── Attachments
      ├── Candidate Timeline
      ├── Resume Skills
      ├── Resume Education
      ├── Resume Employment
      └── Resume Certifications
```

**Primary Relationship Key**

```
CandidateID
```

Every recruitment transaction must reference the CandidateID.

# Database Rules

1. CandidateID is mandatory for all recruitment transactions.
2. CandidateID must never change once assigned.
3. Email EntryID must remain unique.
4. No duplicate CandidateID values.
5. All timestamps use the local system date and time.
6. Every business event creates a Candidate Timeline entry.
7. Every attachment is linked to both CandidateID and Outlook EntryID.
8. Changes to worksheet structures require a version upgrade and release note.
