# Recruitment Automation Suite (RAS)

# MODULE DOCUMENTATION

**Version:** 2.2.0 Baseline

---

# Overview

The Recruitment Automation Suite is organized into independent modules. Each module has a single responsibility and communicates with other modules through public procedures.

---

# Module Architecture

```text
RunRecruitmentAutomation
        │
        ▼
modMain
        │
        ▼
modConfiguration
        │
        ▼
modMailScanner
        │
        ▼
modDispatcher
        │
 ┌──────┼─────────┬──────────┬──────────┬─────────┐
 ▼      ▼         ▼          ▼          ▼         ▼
Resources Offer Interview    BGV     NATS/NAPS Resume
        │
        ▼
Candidate Engine
        │
        ▼
Excel Database
```

# Coding Standards

1. Every module must use `Option Explicit`.
2. Every public procedure must include error handling.
3. No direct worksheet references outside the configuration layer.
4. Use `CandidateID` for all candidate relationships.
5. Log all unexpected errors using `LogError`.
6. Read configurable values from `01_Settings` where applicable.
7. Keep each module focused on a single responsibility.
